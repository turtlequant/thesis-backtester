## 数据源分级与可信度体系

### 目的

建立系统化的数据可信度分级制度，确保分析结论建立在可靠数据之上。未经核查的数据不得用于投资决策，AI计算结果必须经过验证。本算子定义数据源等级、交叉验证规则及数据错误作废机制。

### 核心原则

- 数据质量决定分析质量：垃圾进，垃圾出
- 关键决策数据必须有S级或A级来源支撑
- 单一数据源不可信，核心指标需多源交叉验证
- 基于错误数据的分析必须立即作废

---

### 五级数据源可信度体系

| 等级 | 数据源 | 适用数据类型 | 使用限制 | 信任度 |
|------|--------|-------------|---------|--------|
| **S级（最高）** | 公司年报/业绩公告、交易所官方披露（如港交所披露易、巨潮资讯） | 派息记录、现金余额、负债明细、股本变动、审计报告 | 可作为决策依据，无需额外验证 | 最高 |
| **A级（高）** | 公司官网、投资者关系页面、业绩发布会材料/PPT、公司通函 | 业务数据、战略规划、管理层指引、运营KPI | 可正常用于分析，重大数据建议与S级对照 | 高 |
| **B级（中）** | 专业金融终端（Bloomberg、Wind、Choice、东方财富） | 股价、市值、财务比率、行业对比数据 | 需与S级数据交叉验证后方可用于决策 | 中 |
| **C级（低）** | 财经媒体报道、券商研报、投资社区（雪球/富途等） | 行业趋势、市场情绪、定性判断参考 | 仅供参考，不可作为决策依据 | 低 |
| **D级（不可信）** | AI估算、行业惯例推测、未注明来源的数据 | 无 | 禁止用于投资决策，必须标注"未经验证" | 不可信 |

---

### 数据使用规则

#### 规则一：分析中必须标注数据来源等级

对每个关键数据点，标注其来源等级。格式示例：
- "净利润52亿元 [S级-2024年报]"
- "行业增速约8% [C级-券商研报]"

#### 规则二：决策依据的最低数据等级要求

| 决策类型 | 最低数据等级要求 |
|---------|----------------|
| 买入/卖出建议 | 核心财务数据必须S级，估值指标至少B级（经S级交叉验证） |
| 仓位建议 | 核心数据S级，辅助数据至少B级 |
| 行业判断 | 至少A级，C级仅作辅助 |
| 观察列表纳入 | B级即可，但需标注待验证 |

#### 规则三：交叉验证要求

以下数据必须进行多源交叉验证：

| 数据类型 | 主要来源（S/A级） | 对照来源（B级） | 差异阈值 |
|---------|------------------|----------------|---------|
| 每股派息 | 公司公告/交易所 | 金融终端 | 任何差异需核实 |
| 总负债/有息负债 | 年报资产负债表 | 金融终端 | >5%需核实 |
| 现金及等价物 | 年报/中报 | 金融终端 | >10%需核实 |
| 营业收入/净利润 | 年报/业绩公告 | 金融终端 | >3%需核实 |
| 经营现金流 | 年报现金流量表 | 金融终端 | >5%需核实 |

#### 规则四：数据信任优先级

当不同来源数据冲突时，按以下优先级采信：

```
用户确认的事实 > 公司年报/交易所公告 > 公司官网/IR材料 > 金融终端 > 媒体/研报 > AI估算
```

### 数据错误作废机制

#### 作废触发条件（任一满足即触发）

| 触发条件 | 说明 | 处理方式 |
|---------|------|---------|
| 核心数据错误 | 关键财务数据（如派息、现金、负债）与事实不符 | 立即作废全文分析 |
| 分类判定错误 | 因数据错误导致标的分类/流派判定错误 | 推倒重来，重新判定 |
| 财务数据内部矛盾 | 数据间存在逻辑矛盾（如现金充裕但利息支出畸高） | 暂停分析，核实后再继续 |
| 用户明确指正 | 用户指出数据与事实不符 | 立即采信用户，重新分析 |

#### 作废执行流程

```
步骤1：识别错误
  └── 来源：用户指正 / 交叉验证发现 / 自我审查发现

步骤2：立即停止
  └── 不再基于错误数据给出任何建议
  └── 若已给出买入/卖出建议，立即声明撤回

步骤3：声明作废
  └── 明确说明"基于错误数据，之前的分析结论作废"
  └── 指明错误数据点及其影响范围

步骤4：重新分析
  └── 以正确数据为基准
  └── 重新执行受影响的分析步骤
  └── 重新给出结论和建议

步骤5：记录教训
  └── 记录错误类型和原因
  └── 标记该数据点为"高风险数据点，需强制核查"
```

### AI辅助标签体系

分析过程中，对每个环节标注以下标签以明确人机分工：

| 标签 | 含义 | AI职责 | 人工职责 |
|------|------|--------|----------|
| [COMPUTABLE] | 可计算指标 | 抓取数据、计算比率、生成表格 | 验证数据源、检查异常值 |
| [LLM-REQUIRED] | 需AI分析 | 文本分析、模式识别、趋势判断 | 复核逻辑、确认结论 |
| [MANUAL-ONLY] | 必须人工判断 | 提供计算框架和参考 | 最终判断、决策执行 |
| [CROSS-CHECK] | 需交叉验证 | 提供多源数据对比 | 确认差异原因 |
| [DATA-CONFIDENCE] | 数据置信度标记 | 标记每个数据点的可信度等级 | 核实关键数据 |

### 输出格式

```
数据质量评估：
  - S级数据覆盖率：X%（核心财务指标）
  - 交叉验证完成率：X%
  - 发现数据差异：[列出差异项]
  - 数据质量评级：优 / 良 / 需补充 / 不可用
  - 待验证项：[列出需人工核实的数据点]
```

### 注意事项

- 宁可因数据不足而暂缓分析，也不可基于低质量数据给出结论
- 对于D级数据（AI估算），如必须使用，需明确标注"未经验证，仅供参考"
- 定期检查常用金融终端的数据准确性，发现系统性偏差时调整信任等级
