## 科技股估值方法

科技股估值不能简单套用传统PE/PB框架。高增长期的科技公司往往亏损或微利，PE无意义；资产轻，PB参考价值有限。需要使用PS、EV/Revenue、PEG等更适合成长型企业的估值指标，并结合Rule of 40等行业特有规则。

### 一、PS（市销率）估值

**步骤1：计算当前PS**

```
PS = 总市值 / 最近12个月营业收入（TTM）

从 price_history 获取最新市值
从 income 获取TTM收入（最近4个季度之和）

PS的适用场景：
├── 亏损公司（PE无法使用）
├── 高增长早期公司（利润波动大）
├── SaaS/平台型公司（收入是核心驱动力）
└── 同类公司横向对比
```

**步骤2：PS历史分位数**

```
取近5年PS数据，计算当前PS所处分位：
├── < 10%分位 → 极端低估（或基本面恶化）
├── 10%-25%分位 → 偏低
├── 25%-75%分位 → 合理区间
├── 75%-90%分位 → 偏高
└── > 90%分位 → 极端高估（或增速大幅提升）
```

**步骤3：PS的合理性判断**

```
PS合理范围参考（A股科技）：
├── 高增长(>30%) + 高毛利(>60%)：PS 5-15x 合理
├── 中增长(15-30%) + 高毛利：PS 3-8x 合理
├── 中增长 + 中毛利(30-60%)：PS 2-5x 合理
├── 低增长(<15%)：PS 1-3x 合理
└── 负增长：PS < 1x 可能是合理的

关键原则：PS必须与增速和毛利率匹配
├── 高PS + 高增速 + 高毛利 → 合理
├── 高PS + 低增速 → 高估
├── 低PS + 高增速 → 可能低估（需排除一次性因素）
└── 亏损公司不用PE，但要用PS设定估值锚
```

### 二、EV/Revenue（企业价值/收入）

**步骤1：计算EV**

```
EV = 总市值 + 有息负债 - 现金及等价物

从 price_history 获取总市值
从 balancesheet 获取：
├── 有息负债 = 短期借款 + 长期借款 + 应付债券
├── 现金及等价物 = 货币资金 + 交易性金融资产
│
EV/Revenue = EV / TTM营业收入
```

**步骤2：EV/Revenue vs PS的选择**

```
EV/Revenue 优于 PS 的场景：
├── 公司持有大量现金（EV修正了现金影响）
├── 公司负债率差异大（EV统一了资本结构）
├── 并购估值（EV是收购方真正支付的价格）
│
EV/Revenue的合理范围 ≈ PS的合理范围（微调）：
├── 现金多、负债少 → EV/Revenue < PS（更便宜）
├── 现金少、负债多 → EV/Revenue > PS（更贵）
└── 差异越大，越需要关注资本结构风险
```

### 三、PEG估值（仅限盈利公司）

**步骤1：判断是否适用PEG**

```
PEG适用条件：
├── 最近1年净利润 > 0（亏损公司不适用）
├── 净利润增速 > 0（负增长不适用）
├── 增速相对稳定（大幅波动时PEG失真）
└── PE > 0 且 < 100（极端PE下PEG无意义）
```

**步骤2：计算PEG**

```
PEG = PE(TTM) / 净利润增速(%)

其中：
├── PE = 总市值 / TTM净利润
├── 增速 = 使用3年净利润CAGR（比单年更稳定）
│
PEG判断标准：
├── PEG < 0.5 → 显著低估（如果增速可持续）
├── PEG 0.5-1.0 → 偏低估
├── PEG ≈ 1.0 → 合理估值（彼得·林奇标准）
├── PEG 1.0-2.0 → 偏高估
└── PEG > 2.0 → 明显高估

注意事项：
├── PEG假设增速可持续，需结合增长持续性分析
├── 周期性增长（如一次性订单）不适合用PEG
├── 增速基数效应：低基数导致的高增速会使PEG失真
└── A股科技股PEG中枢通常在1.0-1.5之间
```

### 四、Rule of 40

**步骤1：计算Rule of 40得分**

```
Rule of 40 = 收入增速(%) + 营业利润率(%)

适用于SaaS和平台型科技公司：

├── > 40 → 优秀（增长与盈利平衡好）
├── 30-40 → 良好
├── 20-30 → 一般
├── < 20 → 较差
│
理解Rule of 40的本质：
├── 高增长可以容忍低利润（如增速40%+利润率0%）
├── 低增长必须有高利润（如增速10%+利润率30%）
├── 兼顾增长和利润是最理想的（如增速25%+利润率20%）
└── 低增长+低利润 → 商业模式有问题
```

**步骤2：Rule of 40趋势分析**

```
连续3年数据：
├── 得分逐年上升 → 经营改善
├── 增速贡献 → 利润贡献的转换 → 公司正在成熟化
├── 得分逐年下降 → 警惕
└── 利润率 → 增速的转换 → 牺牲利润换增长（需评估合理性）
```

### 五、DCF敏感性分析

**步骤1：关键假设参数**

```
科技公司DCF的核心变量：
├── 收入增速假设（近5年 → 终值增速）
│   ├── 乐观：当前增速维持3年后逐步回落
│   ├── 中性：增速每年下降3-5pp
│   └── 悲观：增速快速回落至行业均值
│
├── 长期利润率假设
│   ├── 乐观：达到行业龙头水平（如20%-30%营业利润率）
│   ├── 中性：达到行业中位数
│   └── 悲观：利润率无法显著改善
│
├── 折现率
│   ├── 成熟科技：10%-12%
│   ├── 成长期科技：12%-15%
│   └── 早期科技：15%-20%
│
└── 终值增速：2%-4%（不超过GDP增速）
```

**步骤2：敏感性矩阵**

```
构建3×3敏感性表：

              利润率低    利润率中    利润率高
增速低     ┌──────────┬──────────┬──────────┐
           │  最悲观    │  偏悲观    │  中性     │
增速中     ├──────────┼──────────┼──────────┤
           │  偏悲观    │  中性      │  偏乐观   │
增速高     ├──────────┼──────────┼──────────┤
           │  中性      │  偏乐观    │  最乐观   │
           └──────────┴──────────┴──────────┘

当前股价处于哪个区间？
├── 接近最悲观 → 安全边际充足
├── 接近中性 → 合理定价
├── 接近最乐观 → 隐含过高预期
└── 超过最乐观 → 明显高估
```

### 六、与行业中位数比较

**步骤1：多维度对比**

```
对比维度：
├── PS vs 行业中位数PS
│   ├── PS/行业中位数 > 1.5 → 溢价偏高
│   ├── PS/行业中位数 0.8-1.5 → 合理
│   └── PS/行业中位数 < 0.8 → 折价（可能低估或基本面差）
│
├── EV/Revenue vs 行业中位数
├── PEG vs 行业中位数（如适用）
│
└── 溢价/折价的合理性判断
    ├── 增速更高 → 溢价合理
    ├── 毛利率更高 → 溢价合理
    ├── 市场份额领先 → 龙头溢价合理
    ├── 无明显优势但溢价 → 可能高估
    └── 有优势但折价 → 可能是机会
```

### 七、亏损公司估值特别说明

```
亏损科技公司的估值工具选择：
├── PE → 不适用（负值无意义）
├── PB → 参考有限（轻资产公司PB极高）
├── PS → 主要工具（锚定收入规模）
├── EV/Revenue → 主要工具（修正资本结构）
├── DCF → 可用但依赖大量假设
└── 用户数/GMV等运营指标倍数 → 补充参考

亏损公司估值的额外风险：
├── 盈利路径不确定 → 估值区间极宽
├── 现金消耗速度 → 可能稀释股东
├── 竞争格局未定 → 赢者通吃或全行业亏损
└── 建议：给更高安全边际要求（至少30%折扣）
```
