# 投资思路回测引擎 — 产品设计文档

## 一、核心洞察

传统量化回测只能验证可数值化的交易规则（PE<10买入、均线金叉做多），但大量真实的投资决策依赖定性判断：

- "这家公司的高股息能持续吗？"
- "低PE是真便宜还是价值陷阱？"
- "管理层在做价值创造还是资本运作？"
- "行业格局能支撑这个毛利率吗？"

这些判断无法用因子回测，但可以用 **AI盲测回测** 来验证。

**本项目的核心发现：任何可文字描述的投资思路，都可以通过"结构化框架 + 历史截面盲测 + 统计验证"来检验其有效性。**

## 二、已验证的概念证明

### 2.1 实验设计

| 维度 | 设计 |
|------|------|
| 投资思路 | 龟级价值投资（V5.5.6框架）：低PE/PB+高股息+FCF质量 |
| 回测范围 | 2019-06 ~ 2024-12，12个半年截面 |
| 采样方法 | 每截面从龟级筛选池中选5只：最好2+最差2+中间1 |
| 盲测方式 | 隐藏公司名称，仅提供财务数据+行情数据 |
| AI角色 | 标准化分析师——接收相同框架和数据，产出一致可比的判断 |
| 验证指标 | AI评分/建议 vs 实际6个月前向收益 |

### 2.2 核心结果

| 策略 | 样本 | 平均6m收益 | 胜率 |
|------|------|-----------|------|
| 纯量化龟级筛选 | 60只 | +7.5% | - |
| 量化 + AI过滤（仅执行买入） | 9只 | +24.1% | 67% |
| **AI过滤增益** | - | **+16.6pp** | - |

6年累计净值：纯量化 1.50x，高确信AI 2.01x（仅出手8次）。

### 2.3 AI能力边界（关键发现）

| AI强项 | 证据 |
|--------|------|
| 识别高杠杆价值陷阱 | 地产股18只中16只被标记回避，平均收益-10% |
| 高确信买入信号 | ≥70分买入的6只，平均+41.5%，胜率83% |
| 不出手也是alpha | 2019/2024无买入信号=持现金，规避了下跌 |

| AI弱点 | 证据 |
|--------|------|
| 周期底部反转 | 鲁西化工25分→+73%，苏宁环球37分→+136% |
| 牛市中过于保守 | 2020年回避信号平均收益+30.6%（错过牛市） |
| 银行股区分度低 | 银行普遍60-70分，买入/观望界限模糊 |

| 行业 | 样本 | 均收益 | AI效果 |
|------|------|--------|--------|
| 地产/建筑 | 18只 | -10.0% | 最强：16只回避，精准规避陷阱 |

## 三、通用化引擎设计

### 3.1 核心抽象

```
投资思路回测 = f(投资框架, 选股范围, 回测参数)

其中：
  投资框架 = 算子组合（operators/*.md）+ 章节 DAG（strategy.yaml）
  选股范围 = 声明式量化筛选条件（strategy.yaml screening 部分）
  回测参数 = 时间跨度 + 截面频率 + 采样策略 + 验证指标
```

### 3.3 关键组件

#### 组件1：算子注册表 + 框架解析

将投资思路转化为结构化分析框架。

**v6 实现**：
- 分析逻辑以算子（`operators/*.md`）为最小单元，每个算子包含 YAML frontmatter + Markdown 分析指令
- 策略通过 `strategy.yaml` 的 `framework.chapters` 组合算子构建分析框架
- 输出 Schema 从算子 frontmatter 的 `outputs` 字段自动生成
- Prompt 组装在 `runtime.py` 的 `build_system_prompt()` 中内联完成

**通用化方向**：
- 提供算子模版库（价值型/成长型/周期型/事件驱动型...）
- 支持用户创建新算子（新建 `.md` 文件即可）
- 新策略只需创建 `strategy.yaml` 组合已有算子

#### 组件2：量化筛选器

决定"分析哪些股票"。

**v6 实现**：`src/screener/quick_filter.py` — 声明式筛选引擎，读取 `strategy.yaml` 的 `screening` 配置。

**筛选流程**：排除规则 → 预计算因子合并 → 声明式过滤 → 加权评分 → 龟级分级 → Top N

#### 组件3：数据快照器

生成某时间点的完整数据包，确保时间边界不被违反。

**v6 实现**：`src/data/snapshot.py` — `StockSnapshot` 包含行情、财报、治理风险等 15+ 类数据，`ThreadPoolExecutor` 并行加载。

**三层防护**：
1. **数据层硬过滤**：财报按公告日（非报告期）过滤，行情按交易日过滤
2. **Prompt显式注入**：在 system prompt 中明确截止日期和禁止前瞻
3. **盲化处理**：隐藏公司名称和代码，股东匿名化

#### 组件4：AI盲测分析引擎

核心执行层。

**v6 实现**：`src/agent/runtime.py`
- DAG 调度：Kahn 算法拓扑排序，按依赖关系分批执行章节
- 工具沙盒：16 种数据查询工具（OpenAI function calling 格式）
- 行业感知：system prompt 注入行业信息 + 算子级门控
- 输出要求：直接输出分析结论，禁止叙述取数过程

**模型选择**：通过 `strategy.yaml` 的 `llm` 部分配置（model, temperature, max_tokens），支持 OpenAI 兼容 API。

#### 组件5：验证与归因引擎

将AI判断与实际结果对比，产出统计验证报告。

**v6 实现**：
- `src/backtest/outcome_collector.py`：前瞻收益采集（1m/3m/6m/12m + 最大回撤）
- `src/backtest/quality_scorer.py`：5维质量评分（方向40% + 推荐25% + 风险15% + 安全边际10% + 分红10%）
- `src/backtest/crosssection.py`：跨截面纵向对比
- `src/backtest/batch_backtest.py`：批量截面回测

## 五、技术实现

### 5.1 当前技术栈

| 组件 | 技术 | 路径 |
|------|------|------|
| 数据获取 | Tushare Pro API（Provider 抽象） | `src/data/tushare/provider.py` |
| 数据存储 | Parquet（zstd 压缩，月/股票分区） | `src/data/storage.py` |
| 时间点快照 | ThreadPoolExecutor 并行 I/O | `src/data/snapshot.py` |
| 算子注册 | OperatorRegistry（自动 schema） | `src/engine/operators.py` |
| 因子注册 | FactorRegistry（截面+时序） | `src/engine/factors.py` |
| 量化筛选 | 声明式 YAML 筛选 | `src/screener/quick_filter.py` |
| AI分析 | OpenAI 兼容 API（async, tool_use） | `src/agent/runtime.py` |
| 前瞻收益 | 多周期收益 + 回撤采集 | `src/backtest/outcome_collector.py` |
| 质量评分 | 5维加权评分 | `src/backtest/quality_scorer.py` |
| 分析追踪 | SQLite 生命周期管理 | `src/engine/tracker.py` |
| 投研工具 | FastAPI + Vue 3 桌面端（6 页面 + 聊天助手） | `src/desktop/main.py` |

## 六、风险与限制

| 风险 | 缓解措施 |
|------|---------|
| 幸存者偏差 | 采样时纳入已退市/ST股票 |
| 过拟合框架 | 用样本外数据验证；交叉验证 |
| AI模型偏差 | 多模型对比；定期重测 |
| 数据前瞻 | 三层时间边界防护 |
| 采样偏差 | 多种采样策略对比 |
