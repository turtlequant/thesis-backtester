# 投资框架自动进化机制

## 一、核心思路

当前流程是**单向的**：人写框架 → AI执行 → 验证。

进化版流程是**闭环的**：

```
框架V1 → AI盲测 → 验证（对/错归因）
                         ↓
                    AI分析错误模式
                         ↓
                    自动生成改进建议
                         ↓
                    框架V2 → AI盲测 → 验证 → ...
```

这不是简单的参数调优，而是让AI基于回测结果**反思自己的分析方法**，识别系统性盲区，提出框架层面的改进。

## 二、已有的错误模式分析

从V5.5.6的60样本盲测中，已识别以下系统性错误：

### 2.1 假阴性（回避但实际大涨）— 9只，占比15%

| 样本 | AI评分 | 实际收益 | 错误原因 |
|------|--------|---------|---------|
| 苏宁环球 2020-12 | 37 | +136.4% | 2021年初牛市，流动性驱动 |
| 物产中大 2020-12 | 37 | +94.5% | 供应链公司被误判为高杠杆 |
| 滨江集团 2021-12 | 38 | +74.7% | 优质民营房企被行业性否定 |
| 鲁西化工 2020-06 | 25 | +73.3% | 化工周期底部，AI无法预判反转 |
| 中远海控 2023-12 | 52 | +61.7% | 航运周期反弹 |
| 居然智家 2024-06 | 53 | +60.1% | 消费复苏+估值修复 |
| 上海建工 2021-06 | 42 | +29.7% | 建筑央企被高杠杆否定 |
| 保利发展 2021-06 | 42 | +27.0% | 优质央企地产被行业性否定 |
| 中国建筑 2023-12 | 41 | +10.4% | 同上 |

**共性特征**：
1. **周期底部反转**：6/9只是周期股在底部被回避（化工、航运、消费）
2. **流动性牛市**：3/9只集中在2020-12截面（大放水期）
3. **行业"一刀切"**：3/9只是优质央企地产/建筑被行业整体否定

**根因**：框架缺乏宏观环境维度 + 对周期股用了错误的估值方法 + 对央企地产/建筑的行业歧视过度。

### 2.2 假阳性（高分/买入但实际下跌）— 9只，占比15%

| 样本 | AI评分 | 实际收益 | 错误原因 |
|------|--------|---------|---------|
| 方大炭素 2019-12 | 79 | -47.0% | 周期顶部，毛利率开始下滑 |
| 方大特钢 2019-12 | 62 | -48.0% | 同上，钢铁周期拐点 |
| 交通银行 2020-06 | 89 | -13.3% | 银行估值压制，AI高估了低估值的修复动力 |
| 太钢不锈 2021-12 | 62 | -22.7% | 钢铁周期下行 |
| 江苏索普 2022-06 | 68 | -27.6% | 化工周期下行 |
| 兰花科创 2023-12 | 63 | -18.9% | 煤炭景气度下滑 |
| 平煤股份 2024-06 | 68 | -12.2% | 煤炭价格持续下跌 |
| 平煤股份 2024-12 | 68 | -12.8% | 同上 |
| 潞安环能 2024-12 | 79 | -15.0% | 同上 |

**共性特征**：
1. **周期顶部高估**：7/9只是周期股在景气高点，AI看到了好的历史数据但没识别拐点
2. **银行估值陷阱**：1只银行，PE低但市场长期不买账
3. **煤炭景气拐点**：3只煤炭股集中在2024年，景气持续下滑

**根因**：框架没有"周期位置"判断维度 + 对历史高盈利的外推过于乐观 + 缺乏毛利率趋势的拐点识别。

### 2.3 观望区无效（25只，均收益仅+0.1%）

"观望"占比41.7%，说明框架在中间地带缺乏区分力。进一步分析：

- 观望中收益>+20%的：5只（江苏银行+30.7%、华菱钢铁+21.8%等）
- 观望中收益<-20%的：4只（万科-27.5%、江苏索普-27.6%等）
- 大多数在-15%~+15%区间，确实难以判断

**根因**：框架的评分标准在50-70分区间粒度不够，导致大量样本落入"既不够好也不够差"的灰色地带。

## 三、框架改进方向

### 3.1 新增"宏观环境评估"维度

**目标**：解决假阴性中的"周期反转盲区"和假阳性中的"周期顶部高估"。

**机制**：在分析Prompt中注入截面时点的宏观环境摘要，增加一个分析章节：

```markdown
## 第X章：宏观环境与周期位置评估

基于提供的宏观数据：
1. 当前货币政策处于什么阶段？（紧缩/中性/宽松）
2. 该行业在产能周期中的位置？（扩张/峰值/收缩/底部）
3. 当前市场整体估值水平？（高估/合理/低估）
4. 综合判断：宏观环境对该标的是顺风还是逆风？

如果宏观环境为强顺风（如大规模放水期），可以适当放宽安全边际要求。
如果宏观环境为强逆风（如加息周期），应额外收紧安全边际。
```

### 3.2 增强"周期股特殊处理"逻辑

**当前问题**：框架用FCF/PE对周期股估值，但周期股的利润和现金流在不同周期位置差异巨大。

**改进**：

```markdown
## 周期股专项判断

当识别到标的具有强周期特征（毛利率波动>10pp/行业属于周期性行业）时：

1. 不应使用当前利润/现金流作为估值基础
2. 应计算"中周期正常化利润"（取最近5年平均或中位数）
3. 关注"毛利率趋势方向"而非"毛利率绝对水平"
   - 毛利率连续2期下滑 → 可能处于周期顶部，警惕
   - 毛利率连续2期回升 → 可能处于周期底部，机会
4. PB估值优先于PE估值（周期底部利润可能极低但资产价值不变）
```

### 3.3 细化"观望区"的评分标准

**目标**：将50-70分区间的25只"观望"缩减到15只以内，让更多样本获得明确的买入或回避信号。

**方法**：引入"一票决定"规则——

```markdown
## 强制升级/降级规则

观望标的出现以下信号时，强制升级为买入：
- 净现金 > 市值的50%（现金保底足够）
- FCF倍数 < 3倍 + 央国企（极度低估+政府兜底）
- 连续3年分红增长 + 股息率 > 7%（分红确定性高）

观望标的出现以下信号时，强制降级为回避：
- 有息负债 > 市值 + 非央国企（杠杆风险无兜底）
- 经营现金流连续3期为负（利润含金量低）
- 审计意见非标准（财务可信度存疑）
- 大股东质押率 > 50%（资金链风险）
```

### 3.4 央企地产/建筑的特殊处理

**当前问题**：框架对地产/建筑行业"一刀切"回避，但央企地产（保利、中国建筑）和民营地产（华夏幸福、荣盛）的风险完全不同。

**改进**：

```markdown
## 地产/建筑行业特殊处理

1. 如果确认央国企控股：
   - 不自动回避，进入正常分析流程
   - 但额外关注：预收账款趋势、土地储备质量、融资成本
   - 安全边际要求提高（PE<5倍或PB<0.5倍才考虑买入）

2. 如果非央国企：
   - 维持高度警惕
   - 有息负债 > 净资产 → 回避
   - 经营现金流持续为负 → 回避
```

## 四、自动进化流程设计

### 4.1 一次进化迭代

```
Step 1: 收集错误案例
  - 从回测结果中提取所有假阳性和假阴性
  - 附上AI原始分析报告和实际结果

Step 2: AI错误归因
  Prompt: "以下是你之前的分析报告和实际结果。
          请分析每个错误案例的根因：
          - 是数据不足导致的？（缺什么数据？）
          - 是分析框架的逻辑缺陷？（哪个规则需要修改？）
          - 是权重分配不合理？（哪个维度该加权/减权？）"

Step 3: AI生成框架修改建议
  Prompt: "基于以上错误归因，请生成框架V{N+1}的修改清单：
          - 新增的分析维度
          - 修改的评分规则
          - 新增的排除条件
          - 修改的权重分配
          请以可直接执行的Prompt模版格式输出。"

Step 4: 人工审核
  - 检查修改建议是否合理
  - 避免过拟合（只接受有逻辑支撑的修改，拒绝纯数据拟合的修改）

Step 5: 用新框架重跑回测
  - 在相同样本上重跑，对比V{N}和V{N+1}的表现
  - 在样本外数据上验证（避免过拟合）

Step 6: 记录进化历史
  - 保存每个版本的框架、回测结果、改进日志
  - 形成"框架进化谱系"
```

### 4.2 过拟合防护

自动进化最大的风险是过拟合——框架越来越适应历史数据但失去泛化能力。

**防护措施**：

1. **训练/验证集分离**：用2019-2022的数据做框架优化，用2023-2024的数据做独立验证
2. **逻辑审核**：每个修改必须有可解释的投资逻辑，不接受"因为历史数据这样所以这样改"
3. **复杂度惩罚**：框架维度不超过15个，规则不超过30条；每新增1条规则必须删除1条低效规则
4. **多市场验证**：未来扩展到港股/美股后，框架修改需要在多市场上都有效
5. **版本对比**：每次迭代记录完整的版本对比表，一旦新版本样本外表现退化，回退

### 4.3 进化指标

每次迭代应跟踪以下指标：

| 指标 | 目标方向 | V5.5.6基线 |
|------|---------|-----------|
| 方向正确率 | ↑ | 42% |
| 策略增益（alpha） | ↑ | +16.6pp |
| 买入信号胜率 | ↑ | 67% |
| 假阴性率 | ↓ | 15% |
| 假阳性率 | ↓ | 15% |
| 观望占比 | ↓ | 42% |
| 框架复杂度（规则数） | → | ~50条 |

## 五、与数据扩展的协同

框架进化和数据扩展是两个独立但协同的改进方向：

```
           ┌─ 数据扩展 ─→ AI看到更多信息 ─→ 判断更准
           │
改善效果 ──┤
           │
           └─ 框架进化 ─→ AI用更好的方法 ─→ 判断更准
```

**最优实施顺序**：
1. 先做数据扩展（P0），让AI有更多信息可用
2. 再做框架进化，让AI基于更丰富的数据找到更好的分析方法
3. 迭代：数据扩展暴露新的框架盲区，框架改进后又需要新的数据维度

**预期复合效果**：
- 数据扩展单独：正确率42%→48%，增益+16.6pp→+19pp
- 框架进化单独：正确率42%→47%，增益+16.6pp→+18pp
- 两者结合：正确率42%→55%，增益+16.6pp→+25pp
