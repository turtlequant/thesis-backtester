## 二、AI分析中暴露的数据缺口

从60个盲测报告中，AI频繁提到"数据不足"或"无法判断"的场景：

### 2.1 行业与公司身份信息

| 缺口 | AI的表现 | 影响 |
|------|---------|------|
| **行业分类** | AI只能从毛利率/资产结构猜测行业 | 行业判断错误→估值标准选错 |
| **上市时间** | 不知道公司上市多久 | 无法评估"是否经历过完整周期" |
| **主营构成** | 不知道收入/利润来源结构 | 无法判断"多元化vs专注" |
| **所属概念/板块** | 无 | 缺少行业上下游背景 |

**盲测模式处理**：行业分类可以给（不暴露公司名），主营构成需匿名化处理。

### 2.2 行业对比数据

| 缺口 | AI的表现 | 影响 |
|------|---------|------|
| **行业平均PE/PB** | AI无法判断当前估值是行业性低还是个股性低 | 高估了部分银行/钢铁的"低估"程度 |
| **行业平均毛利率** | AI无法判断毛利率是否偏离行业 | 周期股的毛利率判断缺乏锚定 |
| **行业景气指标** | AI完全无法感知行业周期位置 | **最大盲区：周期反转判断失败** |

### 2.3 股东与治理

| 缺口 | AI的表现 | 影响 |
|------|---------|------|
| **实际控制人** | 只能从前十大股东猜测 | 国企/民企判断有时出错 |
| **股东户数变化** | 无 | 缺少筹码集中度信号 |
| **高管持股/增减持** | 无 | 缺少内部人信号 |
| **股权质押** | 无 | 遗漏控股股东资金链风险 |

### 2.4 宏观与市场环境

| 缺口 | AI的表现 | 影响 |
|------|---------|------|
| **宏观经济指标** | AI完全不知道当时的经济环境 | 2020年大放水→AI不知道流动性驱动 |
| **货币政策信号** | 无 | 无法预判周期反转 |
| **市场整体估值** | 不知道全A平均PE/PB水平 | 无法区分"个股便宜"和"市场整体便宜" |
| **行业资金流向** | 无 | 缺少市场情绪维度 |

### 2.5 更精细的财务数据

| 缺口 | AI的表现 | 影响 |
|------|---------|------|
| **分季度数据** | 只有累计值，无法算单季度 | Q4突然变好/变差看不出来 |
| **关联交易** | 无 | 遗漏利润操纵风险 |
| **审计意见** | 无 | 遗漏财务造假风险 |
| **银行专项指标** | 不良率/拨备/净息差/资本充足率 | 银行股分析严重受限 |

## 三、扩展优先级排序

按"AI判断改善效果 / 实现成本"排序：

### P0：高ROI、立刻可做（Tushare已有接口）

| 数据 | Tushare接口 | 预期改善 |
|------|------------|---------|
| **行业分类** | `stock_basic` 中的 `industry` 字段 | 解决行业猜测问题 |
| **行业平均估值** | `daily_basic` 按行业聚合 | 解决"便宜是行业性还是个股性" |
| **股东户数变化** | `pro.stk_holdernumber()` | 增加筹码集中度信号 |
| **主营构成** | `pro.fina_mainbz()` | 了解收入来源结构 |
| **审计意见** | `pro.fina_audit()` | 一票否决财务造假嫌疑 |
| **单季度财务** | 从累计值计算差值 | 看到季度拐点 |
| **上市日期** | `stock_basic` 中的 `list_date` 字段 | 判断公司成熟度 |

### P1：中等ROI、需要一定开发

| 数据 | 来源 | 预期改善 |
|------|------|---------|
| **市场整体估值水平** | `daily_basic` 全市场聚合 → 存入快照 | 解决"市场便宜还是个股便宜" |
| **大盘指数走势** | `pro.index_daily()` 上证/沪深300 | 提供市场环境背景 |
| **股权质押** | `pro.pledge_stat()` | 识别控股股东资金链风险 |
| **高管增减持** | `pro.stk_holdertrade()` | 内部人信号 |
| **银行专项指标** | 需要从年报手工提取或专用接口 | 银行股分析质量提升 |

### P2：高ROI但实现复杂

| 数据 | 来源 | 预期改善 |
|------|------|---------|
| **宏观经济指标** | 央行/统计局数据（M2/社融/PMI/CPI） | **解决最大盲区：周期反转** |
| **行业景气指标** | 行业协会数据、商品价格指数 | 周期股择时能力 |
| **公告文本** | 巨潮资讯/Tushare `pro.anns()` | 重大事件识别 |
| **券商研报摘要** | 需要额外数据源 | 市场预期对比 |

### P3：长期方向

| 数据 | 来源 | 预期改善 |
|------|------|---------|
| **ESG评分** | 第三方数据 | 治理风险维度 |
| **供应链关系** | 企业关联图谱 | 产业链传导分析 |
| **舆情数据** | 社交媒体/新闻 | 市场情绪维度 |
| **港股/美股对标** | 多市场数据源 | 跨市场估值锚定 |

## 四、实现方案

### 4.1 P0 实现（当前sprint）

#### 4.1.1 新增Tushare接口

```python
# tushare_client.py 新增

def fetch_stock_info(ts_code: str) -> pd.DataFrame:
    """获取股票基本信息（行业、上市日期等）"""
    return pro.stock_basic(ts_code=ts_code,
        fields='ts_code,name,industry,market,list_date,is_hs')

def fetch_holder_number(ts_code: str) -> pd.DataFrame:
    """股东户数"""
    return pro.stk_holdernumber(ts_code=ts_code,
        fields='ts_code,ann_date,end_date,holder_num')

def fetch_main_business(ts_code: str) -> pd.DataFrame:
    """主营业务构成"""
    return pro.fina_mainbz(ts_code=ts_code, type='P')  # P=按产品

def fetch_audit_opinion(ts_code: str) -> pd.DataFrame:
    """审计意见"""
    return pro.fina_audit(ts_code=ts_code,
        fields='ts_code,ann_date,end_date,audit_result,audit_agency')

def fetch_pledge_stat(ts_code: str) -> pd.DataFrame:
    """股权质押统计"""
    return pro.pledge_stat(ts_code=ts_code,
        fields='ts_code,end_date,pledge_count,unrest_pledge,rest_pledge,total_share,pledge_ratio')
```

### 4.2 P2 宏观数据实现方案

宏观数据是解决"周期反转盲区"的关键。方案：

#### 4.2.1 预计算宏观快照

为每个截面日期预先准备一份"宏观环境摘要"：

```markdown
## 宏观经济环境（截至2020-06-30）

### 货币政策
- M2同比增速: 11.1%（连续3个月加速，处于2017年以来最高水平）
- 社融存量同比: 12.8%（明显扩张）
- 1年期LPR: 3.85%（年内已下调30bp）
- 信号: 明确的宽松周期

### 经济基本面
- GDP增速: Q1 -6.8%（疫情冲击），Q2 预计转正
- PMI: 50.9（连续4个月在荣枯线上方）
- CPI: 2.5%（温和）
- PPI: -3.0%（工业通缩，上游承压）

### 市场环境
- 上证指数: 2984，年内+3.2%
- 全A平均PE: 18.5（历史30%分位，偏低）
- 全A平均PB: 1.62（历史20%分位，显著偏低）
- 北向资金: 年内净流入+1,200亿（积极）
```

#### 4.2.2 数据来源

| 指标 | 来源 | 获取方式 |
|------|------|---------|
| M2/社融 | 央行 | Tushare `pro.cn_m()` 或手工录入 |
| GDP/PMI/CPI/PPI | 统计局 | Tushare `pro.cn_gdp()` 或手工 |
| LPR | 央行 | 手工录入（变动不频繁） |
| 全A估值 | daily_basic聚合 | 自行计算 |
| 北向资金 | Tushare `pro.moneyflow_hsgt()` | 已有接口 |

#### 4.2.3 实现策略

历史回测只需12个截面的宏观快照，可以手工整理一次性录入（工作量约2小时）。实时运行时再接入API自动获取。

## 五、预期效果评估

| 数据维度 | 预期解决的问题 | 影响的样本 | 预期改善 |
|---------|---------------|-----------|---------|
| 行业分类+行业估值 | AI不再猜错行业，估值锚定更准 | ~10只 | 减少3-5个假阳性 |
| 宏观环境 | 周期反转判断改善 | ~9只(假阴性) | 可能挽回3-5个假阴性 |
| 股东户数 | 增加筹码集中度信号 | ~5只 | 辅助买入时机判断 |
| 审计意见 | 排除财务造假嫌疑 | ~2只 | 避免极端风险 |
| 主营构成 | 多元化/专注判断 | ~5只 | 提升商业模式评估质量 |
| 单季度数据 | 看到拐点信号 | ~8只 | 改善周期判断 |

**综合预期**：如果P0+P2全部实现，方向正确率可能从42%提升到50-55%，策略增益可能从+16.6pp提升到+20-25pp。
